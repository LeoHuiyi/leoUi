leoPlugIn

抽象化来创建有状态的 jQuery 插件

引用：leoUi-tools.js

依赖：jQuery

plugIn 创建jQuery插件

$.leoTools.plugIn( options, [methods] )：$.leoTools.plugIn自动创建统一的jQuery插件

属性：

document: 当前插件的 document

window: 当前插件的 window

$target: 当前插件的选择到的jquery对象

options: 插件实例化后使用选项的对象

version: 插件的字符串版本

选项：

事件：

_init()：

创建插件初始化事件

arguments：

returns：

examples：

                            _init:function(){

                                this.$target.hide();

                            }

_delay( fn [, delay ] )：

在指定延迟后调用提供的函数。保持 this 上下文正确。

arguments：

fn (function | string): 要调用的函数。也可以是插件方法的名称。

delay (number): 调用函数前等待的毫秒数，默认为 0。

returns：

返回 clearTimeout() ID。

examples：

                            this._delay( this._get, 100 );

_on( args [, suppressDisabledCheck ] )：

插件内置绑定事件的方法，同过此方法可：自动添加事件命名空间，调用destroy方法可自动移除事件，设置插件的options.disabledEvent = true可禁用当前实例插件的所有事件，设置了suppressDisabledCheck选项如果事件发生在带有 disableClassName class 的元素上，则不调用事件处理程序。

arguments：

args (): 与jquery中on事件参数一样。

suppressDisabledCheck (string): 参数为'supportDisabled'时，事件发生在带有 disableClassName class 的元素上，则不调用事件处理程序。

returns：

返回 当前插件实例对象

examples：

                            this._on( this.$target, 'click.ok', function(event){

                                $(this).hide();

                            }, 'supportDisabled' );
